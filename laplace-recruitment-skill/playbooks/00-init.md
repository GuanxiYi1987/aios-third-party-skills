# Playbook 00 · 自检 + 引导初始化

触发:自检门失败(`NOTES.md` 缺失 / `init_flag` false / `initialized_skill_version` ≠ 当前版本)。
目标:让 skill 具备运行的初始条件,并把用户策略写进 `NOTES.md`。**此 playbook 是唯一会向用户主动要信息的地方。**

## 铁律:绝不覆盖用户偏好
- `NOTES.template.md` 是模板(只读);用户配置在 `NOTES.md`。
- **首次**(无 `NOTES.md`):从模板生成 `NOTES.md`,逐项引导用户填写。
- **版本迁移**(有 `NOTES.md` 但版本对不上):**只做加法**——对比本版新增了哪些字段/岗位,仅补齐缺失项(给默认值或只就新增必填项问用户),**已有的值一律原样保留**,写完更新 `initialized_skill_version`。
- 任何情况都不得整体重写或清空 `NOTES.md`。

## 首次初始化步骤(逐项确认)
1. **拿 API Key(说人话)**:测试地址已内置,只缺一个 Key。若没有 Key,这样引导用户(别甩环境变量/技术名词):
   > "开始前我需要一个 API Key。请打开招聘系统 →「开放平台」→「API Key」→ 点「创建 Key」→ 把生成的、以 `lap_sk_` 开头的那串复制发给我就行。"
   拿到后 `capabilities` 验连接与 scopes(缺 scope 则说"这个 Key 权限不够,请在开放平台给它勾上 XX 能力后重发",停)。
2. **岗位**:`GET /open/v1/jobs`。若无岗位 → 引导用户创建/确认岗位(`POST /open/v1/jobs`,敏感,需用户确认)。把岗位写入 `NOTES.md` 的 `jobs`。
3. **笔试策略(逐岗位)**:笔试非刚需。逐个问用户"该岗位是否要笔试"→ 写 `needs_written_test`。对 `needs_written_test: true` 的岗位,提醒用户须在「岗位管理」配好笔试题内容(没配则该岗位笔试任务会转人工)。
4. **初筛阈值**:确认 `resume_pass_threshold`(默认 60)。
5. **邮件回溯**:问用户"初始化导入多少天内的邮件"→ 写 `init_backfill_days`;硬顶 `init_backfill_max_emails: 300`。
6. **执行初始回溯导入**(一次性,≤300 封):按 §10 的导入方法,分批 `email-sync-tasks`(每轮 ≤50)直到覆盖所选窗口或达 300 上限,`email-resume-sources/import`(每批 ≤20)导入。**每处轮询都遵守上限**(~5 秒×最多 12 次,见 SKILL.md「轮询必须有上限」),同步任务卡 `pending`/worker 未消费 → 报给用户"同步服务未就绪,稍后重试初始化",别自旋。导入后不管,等系统评分。
7. **SLA / 交接话术**:确认 `sla_*` 天数与 `handoff_*`(按本公司流程,用户填)。
8. **排期意图**:写入 `NOTES.md` 的 `schedule`——**只建 4 个独立自动化定时任务,各自独立汇报/工单,绝不合并**:
   - 频率(全部按用户当地时间):**导入 每天 10:00 / 14:00 / 20:00**、**初筛 每天 11:00 / 15:00 / 21:00**(导入+1h)、**结论 每天 09:00**、**简报-SLA 每天 09:05**。
   - **笔试/面试 = 人工操作,不建定时任务**(HR 经交互模式或小程序发起;playbook 30/40 保留供其调用)。初始化时告知用户这一点。
   - **时区(必问)**:沙箱时间不可信——**向用户确认时区**(默认北京 UTC+8),写入 `timezone`。cron 按用户当地时间表达。
   - **assign(重要)**:每个定时任务都必须 **assign 给招聘 agent(本智能体)** 执行——只有它带着本 skill + API Key 有上下文。交给 FDE 挂 cron 时明确。
   - 实际 cron 挂载 + tz 锚定 + 工单投递由 FDE 自动化 skill 接管(告知用户)。
9. **落标记**:`init_flag: true`、`initialized_skill_version: <当前版本>`、`last_init_at: <now>`。

## 版本迁移步骤(已初始化过、版本变了)
1. 读现有 `NOTES.md`,保留全部已填值。
2. 对比本版 `NOTES.template.md` 的新增字段/新增必填项。
3. 只补缺失:能给默认值的给默认值;必须用户决定的(如新增了"某新岗位要不要笔试")只就该项问用户。
4. 更新 `initialized_skill_version`;其余不动。

## 完成后
输出一份"初始化完成/迁移完成"汇报(已配岗位数、各岗笔试策略、阈值、回溯导入结果、排期意图),交 FDE 投工单。
